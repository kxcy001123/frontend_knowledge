## 本项目同分支开发情况下使用git rebase方式提交

### 常规操作

在同分支开发的情况下，经常在推送代码是，发现已经有人提交了代码，这时需要先拉取代码合并之后才能再次提交并推送，这种操作造成的后果是提交记录不容易看懂。

于是，在同分支开发的情况下，决定使用rebase方式。

### 1. 命令行

无冲突情况下：

1. $ git add .
2. $ git commit
3. $ git push
4. $ /* 如果有人提交过了代码，报错 */
5. $ git pull
6. $ git rebase
7. $ git push

有冲突情况下：

1. $ git add .
2. $ git commit
3. $ git push
4. $ /* 如果有人提交过了代码，报错 */
6. $ git rebase
7. $ /** 解决冲突，保存合并后的文件  */
8. $ git add .
9. $ git rebase --continue
10. $ /* 如果多次提交均有冲突，则重复7,8,9步操作 */
11. $ git push

### 2. sourceTree

1. 点击“拉取”按钮，勾选“用变基代替合并”，点击“确定”
2. 解决冲突
3. 点击“提交”，点击“继续变基”。
